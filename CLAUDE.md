**Spec-first rule:** behavior changes start in the spec doc, never in code. Every behavioral requirement carries an `*Obligation:*` line (its executable test). A clause without an obligation is vague by definition and will be rejected by the spec-lint hook.

**Clause IDs are stable:** never renumber or reuse an ID. New requirements take the next free number in their family. Code, tests, and commits reference clause IDs.

## Conventions (from ADR-0001 / ERD §2, §9)

- TypeScript strict, ESM, Bun ≥ 1.3 (ADR-0003); Bun workspaces: `packages/schemas` (Zod, no internal deps) ← `kernel` ← `agent` (native runtime, AI SDK) ← `cli`; `kernel` ← `cc-plugin`. Kernel never imports `agent` — the api executor is injected via `runEval`'s `extraExecutors` (composition in cli).
- Zod schemas in `packages/schemas` are the single source of truth for types; validate at every storage/IO boundary. Every exported Zod schema is paired with `export type X = z.infer<typeof X>` in the same file — a value export without its type export is incomplete (postmortem: caused a TS2749 typecheck retry; enforced by the schema-consistency gate). Every schema *field* change updates the paired arbitrary in `packages/schemas/test/roundtrip.test.ts` in the same edit — otherwise the round-trip gate fails at gates time, not edit time (postmortem: TraceLink.downstream_hash_at_link).
- SQLite via `bun:sqlite`, hand-written SQL, forward-only numbered migrations. No ORM. A Zod enum that mirrors a SQL `CHECK IN (...)` has a second home the typechecker can't see: widening the enum needs a rebuild migration in the same change, or the INSERT fails at runtime (F-118: Executor gained `api`, migration 0003's eval_run CHECK rejected it). Enforced by the schema-consistency gate — a new enum-mirroring CHECK gets a MANIFEST entry in `scripts/schema-consistency.mjs`, which the gate itself demands.
- ULIDs for events; SHA-256 content hashes for artifacts; money as integer micro-USD; UTC ISO-8601 strings; append-only event tables (no UPDATE). Event-stream queries order by `rowid`, never by ULID or timestamp — both tie within one millisecond (F-060, F-067).
- Tests: `bun test` + fast-check. TUI: OpenTUI. Lint/format: Biome. Obligation tests live at `packages/<pkg>/test/obligations/<CLAUSE-ID>.test.ts` — see the obligation-test skill.
- Comments: only constraints the code can't express. No narration, no doc-comments on internals.
- **Task lists are JSON, always.** Multi-step work is tracked in `.obligato/tasks.json` (committed): `{id, title, state: open|in_progress|completed, clauses[], completed_at}` — mirror of the TEL-7 lifecycle, simplified until Phase 0 builds the real store. Mark tasks `in_progress` when started and `completed` (with timestamp) the moment they finish — never leave state stale, never use markdown checklists for tracking. Edit the board and the findings log with `bun scripts/board.mjs` (validates shape, stamps timestamps, auto-numbers finding IDs) — not hand-written JSON edits; `board.mjs task <id> <state> --title "..."` creates a task that doesn't exist yet (refused when the id exists, active or archived — F-168). `board.mjs archive` moves completed tasks and stamped-closed findings to `.obligato/archive/` (committed) so the active files stay small; miners read both. After a finding-bearing commit, if another commit is imminent this session, let stamp-head's findings.json update ride it instead of a dedicated chore commit — saves a CI run.

**Registry-before-versions rule:** never write a dependency version or tool config from memory — get versions from the registry (`npm view <pkg> version` / `bun pm view`) and scaffold configs with the tool's own init command, then edit; likewise any metadata claim about a repo file (license, version, config value) is read from that file in the same edit — the LICENSE said Apache-2.0 while README/package.json got MIT from memory (F-093). (Postmortem lesson: guessed versions and stale config syntax both failed this way.) This extends to library APIs: AI SDK stream-part shapes, option names, and provider factory signatures are read from the installed `node_modules` `.d.ts` (or official docs) in the same edit, never recalled — the SDK has renamed stream parts across majors. And it includes the contract of any call whose *shape* you are changing (not only new options being added): read the callee's current signature in the installed `.d.ts` before restructuring arguments — ai v7 moved `system:` to `instructions` and rejects system-role entries in `messages`, a change that fails at runtime, not typecheck (postmortem 2026-07-05: 38 tests broke from restructuring the streamText call on the remembered contract).

**LLM-layer test rules (Phases 6–10):** provider/loop tests are fixture-based — recorded stream fixtures, never live endpoints; anything that must hit a real model runs behind an explicit flag, is quarantined out of `bun run gates`, and CI never executes it. Live smoke tests run against Ollama first (free, local); Anthropic runs use subscription quota only — never metered API-key spend without explicit approval.

**Gates:** `bun run gates` runs every gate (doctor → spec-lint → obspec-lint → typecheck → biome → test). CI runs exactly this script. Session hooks (spec-lint/obspec-lint/typecheck on edits, cc-plugin telemetry) fire automatically — but still run `bun run gates` before every commit; hooks check single files, gates check everything. `scripts/doctor.mjs` fails on bun/CI-pin skew and writes the environment manifest to `.obligato/env.json` (proto EVP §4). Run the suite ONCE into a scratch file (`bun run gates > <scratch>/gates.log 2>&1`) and grep/tail that — gates print a trailing summary block, and re-running the ~50s suite just to re-grep a verdict is pure waste (postmortem 2026-07-10: three consecutive full runs for three greps).

**Gotchas (each cost real time once):** run `bunx biome check --write .` before gates — Biome rewraps after every Write, and string replacements miss after a rewrap (re-Read before editing). Bash `cd` persists across calls — stay at repo root. `bun test` exits 1 on an empty suite on Linux only (F-003). `it.todo` needs a function argument to typecheck. After `git push`, fetch the CI run id in a *separate* command before `gh run watch` — querying in the same breath grabs the previous run. Before any paid or quota-burning batch run, smoke-test ONE unit end-to-end through the identical code path (a $0.25 session caught 3 wiring bugs before a 12-session run). Never interpolate content into `sh -c` strings — pass it via env var (backticks in a task statement broke the shell). Container-backed bun tests need explicit timeouts (the 5s default dies on image pull). Scripted string replacements (python `.replace()`, sed) silently no-op on mismatch — every scripted edit must verify its own effect (assert-the-change in the script, or grep immediately after), and assert the occurrence count (`s.count(old) == n`), not just presence — a 1-of-4 partial replace prints 'ok' and ships three stale call sites (F-063, F-085). A soundness fix that changes what a function means makes the deciding parameter REQUIRED — an optional-with-default preserves the old bug at every existing call site, and the required parameter makes the typechecker enumerate the callers for you (F-076→F-085). End-to-end validation runs through the operator-facing surface (the CLI command, not a hand-rolled `bun -e` equivalent) — the hand-rolled path once validated a fix the CLI didn't have (F-085). Batch edit scripts must not abort-and-skip: collect per-edit results and assert the complete list at the end — an abort at edit 1 of 6 silently skipped five edits that were then falsely reported as applied (three phantom fixtures, caught only by re-audit); after any partial batch failure, re-verify every claimed edit before reporting. A callback that ignores its parameters re-evaluates a constant — a '50 consecutive' check returned true after 49 straight failures (F-088); windowed/consecutive conditions need a late-crossing counterexample fixture. A package.json identity or dependency change regenerates and commits bun.lock in the same change — CI runs --frozen-lockfile and fails on drift. Relative imports need an explicit extension (never `./x` — TS2835), and which one is per-package convention: `.ts` in schemas/kernel/agent (`allowImportingTsExtensions`), `.js` in cli — follow the neighboring files, not a global rule. A package's imports must appear in its own `package.json` — workspace hoisting satisfies bun at runtime but tsc fails with TS2307 (bit twice in one session adding zod). The per-edit typecheck hook races multi-edit messages — when two dependent edits ship in one message, the hook failure after the first edit is usually stale; run one `tsc` check after the batch instead of chasing the first hook error (≈6 false alarms in one session); the same race fires cross-agent during parallel slices — same rule, one full `tsc` after all agents land. A test that spawns a child process which calls back into a server hosted by the test process must use async `Bun.spawn` — `spawnSync` blocks the event loop and the pair deadlocks with no error, just a hang. A migration that rebuilds a table (SQLite can't drop NOT NULL in place) trips erd-ripple on the transient `<table>_new` — mention the transient name in the ERD field note, and recreate the table's append-only triggers (DROP TABLE removes them silently; the storage tests catch it). Withholding a credential from a third-party endpoint means substituting a DUMMY that overrides the library's env fallback — the AI SDK reads `process.env.ANTHROPIC_API_KEY` when the passed credential is null, so null/absent leaks the operator's real key; `buildClaudeEnv`'s `obligato-local` is the pattern (F-119, a real leak). A credential-withholding path's obligation test asserts the header VALUE a capture server saw, not the code branch. Making a function async ripples three binding traps into callers, all typechecker-found but expect them: `await f().prop` → `(await f()).prop`; `expect(() => f()).toThrow` → `await expect(f()).rejects.toThrow`; and a helper that calls the now-async fn must itself become async and be awaited at every call site (Phase 7, ~10 rippled test files). Any path used as a DB key or a containment prefix must be realpath-resolved when it can come from a symlinked temp dir or a subprocess's `process.cwd()` — macOS resolves `/tmp` → `/private/tmp`, so a value seeded under one and queried under the other silently misses (bit tool-containment in Phase 6 and trace-link keys in Phase 8, F-124); the agent fixture already `realpathSync`es for this reason. A third-party package whose shipped `.d.ts` fails our strict flags is handled by repo-wide `skipLibCheck` (our own sources stay fully strict, ADR-0003) — don't loosen a source-level flag to accommodate a dependency. When wiring an existing kernel function into a new role, read its full body for **which state each branch mutates**, not just its signature/return — `BudgetMonitor.resolve` clears the pause and grants headroom only for the `continue` action (`escalate`/`block` leave it latched), so a headless path built on `resolveHeadless`'s escalate branch silently killed all further accounting (F-126); pair this with the Phase-8 "does the fn fully discharge its cited clauses" check. Any stochastic loop behavior (bandit exploration's ε-draw) needs an explicit fixture determinism switch or its tests flake intermittently — `RoutingContext.explore` defaults on in production, off in the test helper (F-126 sibling). Mock model tool-call ids must be **globally unique across every model in one session** — the AI SDK matches tool results to calls by id across the whole reconstructed chain, so two mock arrays each numbering `s0,s1` throw `AI_MissingToolResultsError`; prefix ids per model. `appendEvent` treats `parent_id` as the branch point (append-at-arbitrary-parent *is* SES-6 fork), so a test building a *linear* chain must parent each event at `currentHead(listEvents(...))` — parenting at the root or a stale id silently forks the session and reconstruction drops the "other" branch (cost a confusing "session has no user message" in the EVP-10 test). A deterministic reconstruction function (SES-2) that needs a synthetic event id must **derive it from a stable existing id** (flip the first Crockford char of a covered event's ULID), never mint a fresh `ulid()` — a random id breaks identical-chain-identical-context. New CLI output routes through `packages/cli/src/components/sink.ts`'s `write`, never `console.log`/`process.stdout.write` — the UX-9 gate scans `packages/*/src` and only the sink + JSON/stream emitters + cc-plugin protocol writers are allowlisted. Subagents don't inherit the scratchpad dir — an agent prompt that may produce temp/backup files names an explicit out-of-repo temp path, and after any multi-agent edit run, review `git status --short` before `git add -A` (a fixer once left `scratchpad/*.tsx` at repo root, caught only at commit). A "the platform ships this" claim is checked against tsconfig `target`/`lib` — does it *typecheck* — never runtime `typeof`: bun ships JS features ahead of the pinned TS lib (`Set.prototype.union`: runtime yes, ES2023 lib no). Removing a dependency leaves its stale install under node_modules (bun install doesn't prune it) — delete that node_modules entry or fresh-install before trusting local gates; the ui-build gate once passed on a stale typescript 6.0.3 symlink after the dep was dropped (F-156). An operation that rewrites or reorders a previously append-only file (archive, compaction) audits every diff-shape consumer of that file in the same change — stamp-head's added-line heuristic stamped open F-096 after the board archive rewrite (F-169); changelog.jsonl's CI tamper check is the same class. Bash-tool zsh traps: a glob can reach the command unexpanded (`wc docs/specs/*.md` → "open: No such file or directory") — use `find` for file sweeps; a leading-`=` word (`echo ===`) triggers =cmd expansion — quote it. Never construct a commit SHA by string surgery (slice + concat, `cut -c9-`, interpolating a prefix) — a mid-rebase stamp once got a stray char spliced in (`09e56d3f…` for `09e56d32…`), silently overwriting a correct fix_commit; always take the full `git rev-parse <ref>` output verbatim (2026-07-11). Stacked PRs that both append to `.obligato/findings.json`/`tasks.json` conflict at the array tail on every rebase/merge (append-only files, same insertion point) — resolve by **union-by-id** (keep both branches' new rows, dedupe on `id`), never by taking one side, then re-stamp the rebased shas; a plain `--theirs`/`--ours` silently drops a branch's findings (2026-07-11).

## Workflow

1. New feature or behavior change? Run the **feature-pipeline** skill (Obligato's SDLC stages, emulated: ideation → EARS clauses → divergence-test risky clauses → build → verify).
2. Changing behavior? Use the **spec-sync** skill (locate clause → edit spec + obligation → then code). This includes a user-reported bug whose fix changes *observable* behavior — "it's just a fix" / "just a small follow-up" is not an exemption; the clause + obligation come first, and the fix reads back as a `spec_gap` violation otherwise (F-106: repo-first store resolution shipped as prose, no clause, auditor-rejected).
3. Writing tests for a clause? Use the **obligation-test** skill.
4. Mechanical work (renames, formatting, reference updates, changelog lines) → delegate to the **mechanical** agent (Haiku); scouting reads ("where is X defined", "what calls Y", "map this directory") → the **investigator** agent (Haiku, read-only, terse `file:line` table) — both are the routing emulation; don't do either at frontier cost. Threshold: a typechecker-enumerated caller sweep touching ≥ 3 files goes to mechanical; below that, delegation overhead exceeds the savings (postmortem 2026-07-10).
5. Ambiguity suspected in a clause? Run the **divergence** skill (two blind agents, compare readings — SPEC-4 emulated).
6. Before committing a diff that touches behavior, run the **clause-auditor** agent on the *staged* diff, giving it the task context and pointed questions about the riskiest choices — it has found real violations in most audited diffs, including two that would have shipped corrupted telemetry. Every violation/warning it raises — and every spec bug implementation surfaces — gets a row in `.obligato/findings.json` (id, task, source, severity, clauses, summary, root_cause from the file's taxonomy, fix, status) appended in the same commit as the fix. This log is proto-LOOP-1 evidence: the postmortem skill and future benchmark tasks (LOOP-6) mine it. Stage completely and verify the index before messaging the auditor — staging during its run reads as a violation and costs a round. Audits pipeline: stage the batch completely, launch the auditor in the background, and keep building the next batch **unstaged** while it runs — `git diff --cached` stays stable as long as nothing is staged, so audit latency overlaps build time instead of serializing the session (postmortem 2026-07-05: three audits ran this way with zero stalls); never `git add` until the verdict lands. For multi-module phases, run a first audit when the core kernel modules land, not only pre-commit (Phase 4's 8 violations were all present in the earliest modules). A FAIL verdict's **re-audit is delta-scoped**: tell the auditor to verify the named fixes and their clause mapping only and to trust CI for gates/tests (independence holds — CI runs the identical script elsewhere); never scope the *first* audit of a batch — its breadth is what catches false premises (postmortem 2026-07-10: an unscoped re-audit re-ran full suites for 110k tokens, right after the unscoped first audit had caught a false "Phase 4 is unbuilt" premise).
7. **Verification-independence rule:** an exit criterion or manual verification must compute its expected values by a route independent of the implementation under test — different derivation, different boundary definition, or auditor-computed. A check that shares logic with the code it validates is not evidence (postmortem: a self-referential transcript check hid a 3x token over-count, F-031). A new guard's fixture must be discriminating: assert the exact value the fix changes (read the column back) and check what the test reads if the fix is reverted — fixture time-dating once made a survival-clock test unfailable (F-100).
8. Commit spec and code changes together, referencing clause IDs (`feat(kernel): implement TEL-1 step event emission`). Confirm CI with `gh run watch $(gh run list -L1 --json databaseId -q '.[0].databaseId') --exit-status` — never a guessed `sleep`.
9. End of a substantial session → **postmortem** skill (LOOP-1 emulated, propose-only; human is the gate).

## Dogfood telemetry

Hooks append session/tool events to `.obligato/telemetry/events.jsonl` (gitignored, local-first per TEL-2). This is proto-TEL-1: its event shape informs the Phase 0 schema. Known gap discovered by dogfooding: Claude Code hooks expose no token counts — real TEL-1 needs transcript parsing, exactly as the PRD assumes.
