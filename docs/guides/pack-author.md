# Pack Author Guide

A pack is a directory with a `pack.yaml` manifest plus content files. Everything a pack can change about a Obligato session is declared, hashed, signed, and evaluated.

## Layout

```
my-pack/
  pack.yaml          # manifest — required
  pack.sig           # Ed25519 signature over the content hash (releases)
  rules/*.md         # capability: rules
  routing/policy.yaml    # capability: routing-table
  agents/*.yaml      # capability: agent-registry
  suites/**          # capability: eval-suite
  skills/<stage>/**  # capability: stage:<stage>
  context/**         # capability: context-assembly
  README.md, LICENSE, CHANGELOG.md, docs/**   # documentation — no capability needed
```

## Manifest

```yaml
schema_version: 1
name: my-pack            # kebab-case
version: 1.0.0           # semver; obligato pack lint enforces the required bump
kind: efficiency         # stage|efficiency|spec_tooling|routing|eval_suite|agent_registry
kernel_compat: ">=0.1 <2"
capabilities: [rules]    # a CEILING: content addressing an undeclared surface refuses the load
description: One line, said plainly.
```

**Capabilities are enforced by path, not prose** (pack-format §3.1): `routing/**` addresses `routing-table` regardless of what the file says, and a rules file that *talks about* routing does not address it. Unmapped non-documentation paths are refused fail-closed.

## The bar for merging (OSS-4)

1. **Static scan** (SEC-5): your content must pass the injection scanner — no instructions targeting other packs, the gate, telemetry, or exfiltration.
2. **Eval evidence**: an `ablate` run of your pack on the public seed suite, reproducible via its run manifest (EVAL-4), with the ledger entry (`ledger/<pack>/<version>.json`) generated by the runner — never hand-authored (EVP-6 verification will catch it).
3. **Signature** (PACK-2): releases are Ed25519-signed over the content hash; installs verify or refuse. `--unsigned` installs carry an `untrusted` flag.
4. **Version discipline** (PACK-3): capabilities/kind change → major; any content change → minor; metadata only → patch. `obligato pack lint` computes the required bump and CI fails a lower declaration.

The eval gate replaces maintainer taste: a pack that measures `helps` merges; one that measures `hurts` doesn't, whoever wrote it.
